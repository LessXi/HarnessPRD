## Context

HarnessPRD 是一个 AI 驱动的对话式需求工作台，当前前端实现存在以下状态：

**当前架构**：
- 单栏布局，顶部进度条，内容区显示当前阶段
- 按钮分散在各个组件中（App.tsx、DocumentReview.tsx）
- 使用 ViewState 管理 9 个状态：form_editing、ai_dialogue、generating_prd、reviewing_prd、generating_api、reviewing_api、generating_prompts、reviewing_prompts、completed
- 数据存储在 localStorage 的单键 `harnessprd:project` 中

**问题**：
1. 用户需要寻找操作按钮，流程引导不清晰
2. 无法回退到之前步骤，只能重新开始
3. 每个阶段都需要手动推进，自动化程度不够
4. 完成后没有统一的成果展示页面

## Goals / Non-Goals

**Goals:**
- 统一管理页面跳转逻辑，用户通过按钮就能走完整个流程
- 每个阶段只显示允许的操作，减少用户决策负担
- 支持回退到任意已完成步骤，保留后续数据
- 半自动推进，阶段完成后提示用户是否继续
- 成果汇总页，统一展示和下载所有生成文档

**Non-Goals:**
- 不实现云端保存或分享功能
- 不实现多语言支持
- 不实现主题切换
- 不添加 `summary_review` 状态

## Decisions

### 1. 侧边栏布局

**决策**：采用侧边栏 + 内容区布局

**理由**：
- 侧边栏可以固定显示进度条和操作按钮，用户随时可见
- 内容区专注显示当前阶段内容，减少干扰
- 移动端可以折叠为抽屉式，保持响应式

**替代方案**：
- 顶部导航栏：空间有限，操作按钮可能被截断
- 底部固定栏：移动端键盘弹出时会被遮挡

### 2. 状态扩展

**决策**：扩展 ProjectState，添加 completedSteps 和 autoAdvance 字段

**理由**：
- completedSteps 记录已完成步骤，支持回退功能
- autoAdvance 控制是否自动推进，支持半自动模式
- 保持向后兼容，旧数据自动迁移

**数据结构**：
```typescript
interface ProjectState {
  // ... 现有字段
  completedSteps: ViewState[]  // 已完成步骤列表
  autoAdvance: boolean         // 是否自动推进
  pendingUpdates: ViewState[]  // 待更新步骤列表
}
```

### 3. 回退策略

**决策**：回退时保留后续数据，标记为待更新

**理由**：
- 避免数据丢失，用户可能想参考之前生成的内容
- 待更新状态提醒用户数据可能不一致
- 用户可以选择重新生成或跳过

**替代方案**：
- 回退时清空后续数据：可能导致用户丢失重要工作
- 回退时询问用户：增加操作复杂度

### 4. 半自动推进

**决策**：每个阶段完成后显示提示栏，用户选择是否继续

**理由**：
- 不打断用户工作流程（相比模态对话框）
- 用户可以选择继续、跳过或返回
- 支持取消正在进行的生成过程

**交互设计**：
- 底部提示栏显示"是否继续生成下一个文档？"
- 按钮：继续、跳过、返回
- 生成过程中显示取消按钮

### 5. 成果汇总页

**决策**：完成阶段改为成果汇总页，展示所有文档

**理由**：
- 统一展示所有生成文档，方便用户查看和下载
- 支持预览、单独下载、打包下载
- 提供"开始新项目"按钮

**功能**：
- 文档列表：PRD、接口文档、提示词套件
- 每个文档：预览、下载、复制按钮
- 一键下载全部（ZIP）
- 项目信息：会话ID、创建时间、完成时间

### 6. ZIP 下载实现

**决策**：使用 JSZip 库在前端打包

**理由**：
- 与无状态后端架构一致，所有数据在前端
- 不需要后端支持，减少服务器负载
- JSZip 是成熟的库，支持 ZIP 格式

**替代方案**：
- 后端打包：需要上传数据到服务器，增加复杂度
- 逐个下载：用户体验差

## Risks / Trade-offs

### 风险 1：状态复杂度增加
**风险**：添加 completedSteps、pendingUpdates 等字段，状态管理变复杂
**缓解**：
- 使用 TypeScript 严格类型检查
- 编写状态转换验证函数
- 添加单元测试覆盖状态转换逻辑

### 风险 2：回退导致数据不一致
**风险**：用户回退修改对话后，后续文档可能不准确
**缓解**：
- 标记后续文档为待更新状态
- 提供重新生成功能
- 在 UI 中明确提示数据可能不一致

### 风险 3：移动端适配复杂
**风险**：侧边栏在移动端需要折叠为抽屉式，增加适配工作
**缓解**：
- 使用 Tailwind 的响应式工具类
- 优先实现桌面端，再适配移动端
- 考虑使用 shadcn/ui 的 Sheet 组件

### 风险 4：性能影响
**风险**：侧边栏渲染可能影响页面性能
**缓解**：
- 使用 React.memo 优化渲染
- 侧边栏内容按需加载
- 避免不必要的重渲染

## Open Questions

1. **文档复杂度如何计算？**
   - 选项：字数、段落数、标题层级深度
   - 建议：使用字数作为主要指标，段落数作为辅助

2. **待更新状态如何显示？**
   - 选项：⚠ 图标、不同颜色、文字提示
   - 建议：使用 ⚠ 图标 + 文字提示

3. **取消生成后如何恢复？**
   - 选项：保留部分内容、完全丢弃、提示用户
   - 建议：保留已生成的部分内容，提示用户可以继续生成
