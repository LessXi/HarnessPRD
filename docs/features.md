# HarnessPRD 功能清单

> 版本 1.0 · 归档时间：2026-06-14

**优先级定义：**
- **P0** = 核心功能，没有它产品跑不起来
- **P1** = 重要功能，影响核心体验
- **P2** = 锦上添花，可以后做

---

## 步骤一：填写产品信息（7 点）

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| 1.1 | 表单页面 | P0 | 独立路由页面，展示产品信息表单 |
| 1.2 | 文本输入字段 | P0 | 产品名称、一句话定义、核心痛点、目标用户（4 个文本域） |
| 1.3 | 动态功能列表 | P0 | 输入框 + 添加按钮 → 标签列表；每条可删除；至少 3 条才可提交 |
| 1.4 | 可选扩展字段 | P2 | 技术限制、时间预期、竞品（3 个可折叠文本域） |
| 1.5 | 表单校验 | P0 | 必填检查 + 列表条数检查 + 错误提示 |
| 1.6 | 提交按钮 | P0 | 提交表单 → 创建 Session → 跳转到对话页 |
| 1.7 | Session 存储 | P0 | 后端 UUID 创建会话，表单数据存内存 dict |

## 步骤二：AI 对话澄清（12 点）

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| 2.1 | 对话页面 | P0 | 独立路由页面，展示历史消息 + 输入框 |
| 2.2 | AI 首轮追问 | P0 | 进入对话页自动触发，基于表单数据发出第一个追问 |
| 2.3 | SSE 流式接收 | P0 | 后端调用 LLM → SSE 逐 token 返回前端 |
| 2.4 | 打字机效果 | P1 | 前端逐字渲染 AI 回复 |
| 2.5 | 对话上下文管理 | P0 | 后端维护 messages[]，每次请求追加历史 |
| 2.6 | 消息气泡展示 | P1 | 用户左对齐、AI 右对齐 + 头像区分 |
| 2.7 | 对话历史 | P0 | 页面内可滚动查看完整对话 |
| 2.8 | 输入框 + 发送 | P0 | 用户输入 + 回车/按钮发送 |
| 2.9 | 需求摘要生成 | P0 | 对话结束时 AI 提炼为结构化 JSON |
| 2.10 | 摘要确认卡片 | P0 | 展示需求摘要，用户点"确认"或"继续补充" |
| 2.11 | "进入生成"按钮 | P0 | 确认摘要后出现，进入文档生成阶段 |
| 2.12 | 加载状态 | P1 | 等待 AI 回复时显示加载动画 |

## 步骤三：文档生成（24 点）

### 3a. 生成 PRD（7 点）

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| 3a.1 | PRD 生成页面 | P0 | 独立页面，进入即启动生成 |
| 3a.2 | Write 阶段 | P0 | 基于需求摘要生成初版 PRD，SSE 流式展示 |
| 3a.3 | Review 阶段 | P1 | 审核 Agent 输出问题清单 |
| 3a.4 | Rewrite 阶段 | P1 | 按审核意见修改 |
| 3a.5 | 循环控制 | P1 | Review → Rewrite 轮次管理，上限 3 轮 |
| 3a.6 | 循环过程可视化 | P2 | 展示每轮审核意见 + 修改内容 |
| 3a.7 | PRD 完成确认 | P0 | 用户确认后进入下一阶段 |

### 3b. 生成接口文档（7 点）

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| 3b.1 | 接口文档生成页面 | P0 | 独立页面，进入即启动生成 |
| 3b.2 | Write 阶段 | P0 | 基于需求摘要 + 最终 PRD 生成，SSE 流式展示 |
| 3b.3 | Review 阶段 | P1 | 审核与 PRD 的一致性 |
| 3b.4 | Rewrite 阶段 | P1 | 按审核意见修改 |
| 3b.5 | 循环控制 | P1 | Review → Rewrite 轮次管理 |
| 3b.6 | 循环过程可视化 | P2 | 展示每轮过程 |
| 3b.7 | 接口文档完成确认 | P0 | 用户确认后进入下一阶段 |

### 3c. 生成提示词套件（7 点）

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| 3c.1 | 提示词套件生成页面 | P1 | 独立页面，进入即启动生成 |
| 3c.2 | Write 阶段 | P1 | 基于需求摘要 + PRD + 接口文档生成 |
| 3c.3 | Review 阶段 | P2 | 审核提示词一致性 |
| 3c.4 | Rewrite 阶段 | P2 | 按审核意见修改 |
| 3c.5 | 循环控制 | P2 | Review → Rewrite 轮次管理 |
| 3c.6 | 循环过程可视化 | P2 | 展示每轮过程 |
| 3c.7 | 全部完成 | P1 | 三份文档全部生成完成 |

### 3d. 文档操作·通用（3 点）

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| 3d.1 | 编辑文档内容 | P1 | 确认页提供 Markdown 编辑器，用户可修改 AI 生成内容 |
| 3d.2 | 复制文档内容 | P1 | 一键复制当前文档全文到剪贴板 |
| 3d.3 | 下载文档 (.md) | P1 | 下载当前文档为 `.md` 文件 |

---

## 优先级汇总

| 优先级 | 数量 | 范围 |
|--------|------|------|
| **P0** | 20 点 | 表单核心字段 + 校验 + 提交（步一）；对话页 + SSE + 上下文 + 摘要确认（步二）；PRD/接口文档的 Write + 确认（步三） |
| **P1** | 14 点 | 打字机效果、消息气泡、加载状态（步二）；PRD/接口文档 Review+Rewrite+循环（步三）；提示词套件 Write+完成（步三）；编辑/复制/下载文档（步三） |
| **P2** | 9 点 | 可选扩展字段（步一）；全部循环可视化（步三）；提示词套件 Review+Rewrite+循环（步三） |

**总计：43 个功能点**
